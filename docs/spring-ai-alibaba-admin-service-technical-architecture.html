Spring AI Alibaba Admin Service - Technical Architecture

Technical profile: concrete modules, Spring Boot runtime, API surfaces, middleware, data stores, telemetry, and model-provider flows

Backend Surface

  • • Boot app: spring-ai-alibaba-admin-server-start
  • • Product APIs: prompt, dataset, evaluator, experiment, observability
  • • Builder APIs: apps, workflows, tools, MCP, graph-studio DSL/run
  • • Runtime OpenAPI: chat/workflow completions with streaming support

State And Middleware

  • • MySQL stores admin entities via MyBatis/JPA/Druid
  • • Redis stores cache and workflow task context through Redisson
  • • Elasticsearch backs RAG vectors and LoongSuite trace indices
  • • RocketMQ drives async document indexing into the search plane

Integration Notes

  • • Nacos is the config and prompt-proxy bridge for connected agent apps
  • • LoongCollector receives OTLP traces on port 4318 and writes to Elasticsearch
  • • Model configuration templates cover OpenAI, DashScope, and DeepSeek
  • • Compose exposes frontend:80, backend:8080, and middleware service ports
