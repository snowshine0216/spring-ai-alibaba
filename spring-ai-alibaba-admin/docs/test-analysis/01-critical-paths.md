## CP-6: User login

- **Start**: `POST /console/v1/auth/login` (`AuthController`)
- **Key nodes**:
  - `AuthController.login` â†’ `AuthService.login(LoginRequest)` â€” verifies password (`PasswordCrypt`, see existing `PasswordCryptTest.java`)
  - Issues access + refresh tokens (`TokenResponse`); pairs with `POST /console/v1/auth/refresh-token` and `POST /console/v1/auth/logout`

- **End / success**: `Result<TokenResponse>` with valid access + refresh JWTs; failure returns non-zero `code` per the integer-coded `Result` wrapper
- **Why critical**: rubric 4 (tenancy boundary â€” every downstream `workspace_id` scoping starts here); rubric 6 (auth is bug-magnetism by nature â€” token expiry, refresh races, password hashing changes)
